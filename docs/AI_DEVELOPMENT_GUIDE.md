# 自律型AIを活用したアプリ開発プロセス（BrainDump版）

このドキュメントは、BrainDump プロジェクトにおいてAIエージェントを活用し、品質を担保しながら安全かつ効率的にアプリを開発・運用していくためのガイドラインです。

---

## 全体フロー

1. **【上流】要件定義** （人間が `docs/spec.md` に意図を記述）
2. **【中流】テスト設計** （AIが要件に基づき、E2Eテストまたは単体テストを作成）
3. **【中流】実装と検証** （AIが実装し、テストをパスさせ、視覚的に報告）
4. **【下流】最適化** （AIがコードとコストの最適化・リファクタリングを実施）

---

## 1. 【要件定義】意図の明文化

- すべての機能追加・変更は `docs/spec.md` を起点とします。

## 2. 【テスト設計】ゴールポストの設定

- 実装の前に、AIに成功条件（テストコード）を定義させます。
- Frontend: Playwright / Vitest
- Backend: pytest

## 3. 【実装と視覚的検証】

- 実装完了後、内蔵ブラウザ等を利用して動作確認を行い、スクリーンショット等でエビデンスを提示します。
- 特に「入力時の低刺激なUX」と「チャット時の高精度な回答」を確認します。

## 4. 【最適化】

- `Gemini.md` のルールに従い、Pythonicなコード、および効率的なFirestoreクエリにリファクタリングします。
- AI API (Gemini/Whisper) のトークン使用量やコスト効率にも配慮します。
