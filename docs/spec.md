# BrainDump External Memory - アプリケーション仕様書 (Spec)

## 1. 開発の目的 (Purpose)

本システムの主眼は、人間の脳における「情報保持」と「情報処理」の役割を分離し、デジタルな外部メモリへ委ねることで、ユーザーの精神的リソースを解放することにある。

- **認知負荷の軽減 (Cognitive Offloading)**:
  「忘れてはいけない」という脳の防衛本能（ツァイガルニク効果）を解除するため、信頼性の高い記録場所を提供し、作業記憶（ワーキングメモリ）の空き容量を確保する。
- **入眠障害の改善**:
  就寝前の反芻思考（ループする悩みやTODO）を物理的に出力させることで、脳を「覚醒モード」から「休息モード」へ切り替えるトリガーとする。
- **自己客観視の促進 (Self-Reflection)**:
  蓄積された主観的な思考を、AI（RAG）を介して客観的なデータとして再提示することで、ユーザー自身の思考パターンの発見やメタ認知を支援する。
- **非干渉な記録体験**:
  記録時にAIが即座に反応しない設計により、ユーザーが他者の目を気にせず、最も純粋で未加工な思考（生データ）を吐き出せる環境を構築する。

## 2. ユースケース (Use Cases)

### ユースケース A：就寝前のマインド・クリアリング

- **状況**: 布団に入った後、翌日の不安や今日の反省が止まらず眠れない。
- **アクション**: スマホを取り出し、アプリの録音ボタンを押して頭にあることをすべて口に出す（または入力する）。
- **結果**: 画面がクリアになり「保存完了」を確認することで、脳が「この情報は記録された」と判断し、思考のループが停止。スムーズな入眠につながる。

### ユースケース B：感情の爆発・パニック時のデトックス

- **状況**: 強いストレスや怒りを感じ、冷静な思考が困難な状態。
- **アクション**: 支離滅裂な状態のまま、感情をそのまま音声で叩き込む。AIの反応を待つ必要がないため、遮られることなく吐き出し切る。
- **結果**: カタルシス効果により感情が鎮静化される。データはバックグラウンドでタグ付けされ、後日冷静な時に振り返ることが可能になる。

### ユースケース C：散らばったアイデアの断片回収

- **状況**: 移動中や作業の合間に、断片的なアイデアが次々と浮かぶが、整理する時間がない。
- **アクション**: 箇条書きや音声で、脈絡なくアイデアをアプリに放り込む。
- **結果**: 脳の「忘れないように保持する」エネルギーを消費せずに済む。後からキーワード検索やAI要約を通じて、バラバラのアイデアを統合できる。

### ユースケース D：週次・月次のセルフ・リフレクション

- **状況**: 自分のメンタル傾向や、最近何に時間・意識を奪われているかを確認したい。
- **アクション**: AIチャット（振り返りモード）を起動し、「最近、僕が一番気にしていることは何？」「1ヶ月前と比べて悩みは変わった？」と質問する。
- **結果**: RAGにより過去の全記録から抽出された「事実」に基づき、AIが自分の傾向を提示. 主観的な思い込みではない、データに基づいた自己理解が可能になる。

## 3. 機能要件 (Functional Requirements)

### 3.1 入力管理 (Input Management)

- **ハイブリッド・エディタ (Drafting Area)**:
  - 音声入力による文字起こし結果と、キーボードによるテキスト入力が同一のエディタ内にリアルタイムで統合される。
  - ユーザーは送信前に、文字起こしされた内容を自由に手動修正（デリート、追記）できる。
- **音声入力の挙動 (Local STT/Streaming)**:
  - 録音ボタンを押している間、またはトグルがオンの間のみ、音声をテキストに変換してエディタに追記していく（※現在は Gemini Flash-Lite を用いたバックエンド処理で実現）。
  - この段階では「一時的なテキストデータ」として扱い、バックエンドの長期記憶DBには書き込まない。
- **送信トリガー (The "Dump" Action)**:
  - ユーザーが明示的に「送信ボタン（Dumpボタン）」を押した瞬間に、初めてAPIエンドポイント（`POST /api/dump`等）へデータが送出される。
  - 送信完了と同時に、フロントエンドの入力バッファを完全にクリアし、視覚的な「空っぽになった」状態を作る。
- **ユーザーインターフェースのステート管理**:
  1.  **待機（Idle）**: 入力欄が空の状態。
  2.  **編集中（Drafting）**: 音声やテキストが入力されているが、まだ送信されていない状態。（誤操作防止のため、画面を閉じようとした際に「内容は保存されませんがよろしいですか？」という警告を出す）
  3.  **送信中（Syncing）**: 送信ボタン押下後、バックエンドへのリクエストが完了するまでの状態。
  4.  **完了（Cleared）**: 入力欄が白紙に戻り、情報が外部化された状態。

### 3.2 AIバックエンド処理 (AI Processing)

- **自動構造化 (Processing)**: Gemini 3.1 Flash 等を使用し、以下のメタデータを抽出。
  - **Topic**: カテゴリ分類タグ（例：#仕事、#不安、#アイデア）。
  - **Sentiment**: 感情トーンの数値化（ポジティブ・ネガティブ・中立）およびラベル。
  - **Actionable**: 重要度判定（タスクが含まれているか）。
  - **Summary**: 短い要約。
- **長期記憶化 (Embedding)**:
  - 構造化データをベクトル化し、Firestore Vector Search に保存。
  - 過去の類似データとの関連性を計算し、検索性を向上させる。

### 3.3 履歴・管理機能 (History & Management)

- **アーカイブ閲覧**:
  - **タイムライン表示**: 投稿日時の降順による一覧表示。
  - **動的フィルタリング**: 日付、タグ、感情スコアによる絞り込み。キーワードによる全文検索.
  - **詳細表示**: AI要約・タグと、元の生データ（Raw content）の切り替え閲覧。
- **データコントロール**:
  - **編集・削除**: 保存済みデータの修正、および物理削除機能。
  - **エクスポート**: 全データを Markdown または JSON 形式で一括ダウンロード。

### 3.4 振り返り機能 (Reflection & RAG)

- **オンデマンド・フィードバック**:
  - ユーザーが明示的にリクエストした時のみAIチャットが起動。
  - 過去の全記録から関連コンテキストを抽出し、Gemini 3.1 Pro がユーザーの過去データのみを根拠とした客観的な回答を生成。
- **インサイト分析**:
  - **傾向レポート**: 週次・月次で頻出するトピックや感情推移の可視化。
  - **セルフ・リフレクション支援**: 過去の発言を引用し、思考パターンの気づきを提供。

### 3.5 システム・UI要件 (System & UI)

- **表示・設定**:
  - **フォーカス・モード**: 入力ボタン以外のUI要素を非表示にする設定。
  - **ナイトテーマ**: 就寝前を想定し、コントラストを抑えたダークモードを標準化。
- **セキュリティ**:
  - **認証機能**: 生体認証（FaceID/指紋）またはパスコードによるアプリロック。Firebase Auth との完全統合.
  - **データ保護**: 通信および保存データの暗号化。

## 4. 技術スタック

- **Frontend**: Next.js 16+ (App Router), Tailwind CSS, shadcn/ui
- **Backend (AI Engine)**: FastAPI (Python 3.10+)
- **Database/Auth**: Firebase (Firestore, Auth, Storage)
- **Vector Search**: Firestore Vector Search (Firebase Extensions)
- **AI Models (Latest Gemini 3.1 Series)**:
  - **Speech-to-Text**: Gemini 3.1 Flash-Lite
  - **Structuring**: Gemini 3.1 Flash
  - **Analysis/RAG**: Gemini 3.1 Pro
  - **Embedding**: gemini-embedding-001 等

## 5. データ構造 (Firestore Schema)

### Collection: `entries`

```typescript
{
  id: string,
  user_id: string,
  content: string,        // 原本（テキストまたは文字起こし結果）
  summary: string,        // Geminiが生成した要約
  topic: string[],        // カテゴリタグ
  sentiment: {
    score: number,        // -1.0 to 1.0
    label: string
  },
  is_actionable: boolean,
  created_at: Timestamp,
  embedding: number[]     // ベクトルデータ（768次元など）
}
```
